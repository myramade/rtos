How RMS and EDF Work

Rate Monotonic Scheduling

RMS assigns static priorities inversely proportional to task period. The task with the shortest period always has the highest priority and can preempt any longer-period task at any time.

The Liu & Layland utilisation bound guarantees that any task set with U ≤ n(2^(1/n)−1) is schedulable under RMS. At n = 1 the bound is 100%; as n grows it converges to ln 2 ≈ 69.3%.

Earliest Deadline First

EDF assigns dynamic priorities: at every scheduling point the ready task with the nearest absolute deadline runs first. If a new task releases with an earlier deadline than the currently running task, it immediately preempts it.

EDF is optimal for preemptive uniprocessor scheduling: any task set with U ≤ 1 is feasible under EDF. The cost is higher runtime overhead and more complex implementation compared to RMS.
