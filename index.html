Quick Reference

RMS — Rate Monotonic Scheduling

  • Static priority based on period
  • Shorter period → higher priority
  • Optimal among fixed-priority algorithms
  • Schedulability: U ≤ n(2^(1/n)−1)
  • At n→∞, bound approaches ln(2) ≈ 69%

EDF — Earliest Deadline First

  • Dynamic priority based on absolute deadline
  • Task with nearest deadline runs first
  • Optimal for preemptive uniprocessor
  • Schedulability: U ≤ 1 (100%)
  • More complex to implement than RMS

WCET Response Time Analysis

  • R⁽⁰⁾ = C (own WCET)
  • R⁽ᵏ⁺¹⁾ = C + Σⱼ∈hp ⌈R/Tⱼ⌉×Cⱼ
  • Iterate until R⁽ᵏ⁺¹⁾ = R⁽ᵏ⁾
  • Task feasible if R ≤ T
  • Accounts for all preemptions by hp tasks

Key Security Threats

  • Priority inversion → PIP / PCP
  • Stack overflow → canary + MPU
  • Race conditions → mutex / semaphore
  • Timer overflow → use uint32_t
  • IPC buffer overflow → bounds checking
